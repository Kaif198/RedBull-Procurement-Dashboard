# Red Bull Quality Performance Procurement Dashboard

A comprehensive, executive-level procurement and supply chain intelligence dashboard built with React and Vite. It is designed to track KPIs, analyze vendor performance, manage contracts, simulate market conditions, and monitor global risks, all while adhering to Red Bull's premium, high-energy brand aesthetics.

## Key Modules & Features

- **Overview (Command Center):** High-level view of Total Spend, Active Suppliers, Global Compliance, and YTD Savings. Features a master procurement map and top-level KPI tracking.
- **Category Intelligence:** Deep dive into specific procurement categories (e.g., Marketing, Logistics, Packaging). Analyzes spend distribution and category-specific compliance.
- **Vendor Management:** Detailed breakdown of supply chain partners. Includes vendor scoring, ESG (Environmental, Social, and Governance) sustainability tracking, and performance metrics.
- **Contract Lifecycle:** Tracks active agreements, upcoming renewals, and negotiated savings. Emphasizes risk and compliance at the contract level.
- **Global Risk Monitor:** Real-time tracking of geopolitical, environmental, and financial risks impacting the supply chain. Features heatmaps and risk mitigation narratives.
- **Market Intel & Strategy Simulator (Sandbox):** An interactive What-If P&L Stress-Test Sandbox. Allows users to model freight cost volatility, inflation drift, and forex exposure (EUR/USD) to instantly calculate potential ROI for strategic sourcing initiatives.

## Technology Stack

- **Frontend Framework:** React 18, Vite
- **Routing:** React Router DOM
- **Data Visualization:** Chart.js, react-chartjs-2
- **Icons:** Lucide React
- **Styling:** Custom Vanilla CSS with Red Bull brand design system (Glassmorphism, Neon Accents, Dark Theme)

## Getting Started

### Prerequisites
- Node.js (v16 or higher recommended)

### Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/Kaif198/RedBull-Procurement-Dashboard.git
   ```
2. Navigate to the frontend directory:
   ```bash
   cd "RedBull-Procurement-Dashboard/frontend"
   ```
3. Install dependencies:
   ```bash
   npm install
   ```
4. Start the development server:
   ```bash
   npm run dev
   ```
